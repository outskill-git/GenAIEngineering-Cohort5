Invoice
This table contains invoice information, including unique identifiers, customer details, billing addresses, and total costs. Each row represents a single invoice, with the InvoiceId serving as the primary key to distinguish between them. The table provides a comprehensive overview of invoice data, including when it was issued, where it was billed, and the total amount due.

The InvoiceLine table contains details of individual items within an invoice, with each row representing a unique line item. It captures information such as the invoice it belongs to, the specific track or product, its unit price, and the quantity purchased. The table is linked to invoices through the InvoiceId column, allowing for the breakdown of invoice totals into component parts.

Playlist
The Playlist table contains information about music playlists, with each playlist uniquely identified by a PlaylistId. This table serves as a repository for playlist metadata, providing a foundation for managing and organizing music collections.
PlaylistTrack
This table contains many-to-many relationships between playlists and tracks. It serves as a bridge table, allowing multiple tracks to be associated with multiple playlists. The PlaylistId and TrackId columns, which are both primary keys, uniquely identify each playlist-track combination.
